# 1e4?

In my code, I used 1e4 instead of 10000 because 1e4 is scientific notation in JavaScript that represents 10,000. It's a shorthand way of writing large numbers, where "e" stands for "exponent" and 1e4 means 1 × 10^4, which equals 10,000. Using 1e4 makes the code slightly more concise and is commonly used in programming for readability, especially with larger numbers. Both 1e4 and 10000 are functionally the same; it's mostly a matter of style and preference.

# i % 15 ?

In mathematics, when we want to find a number that is divisible by both 3 and 5, we use the concept of the least common multiple (LCM). The LCM of 3 and 5 is 15, so any number that is divisible by both 3 and 5 will also be divisible by 15. That's why we can simply check i % 15 === 0 instead of checking i % 3 === 0 and i % 5 === 0 separately—it's a quicker and cleaner way to get the same result.
